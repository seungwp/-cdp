# 2026 융합 Capstone Design Project

## 운전자 위급상황 대응을 위한 V2V 연계형 갓길 대피(MRM) 시스템

---

## 팀 구성 및 역할

| 이름 | 역할 | 담당 폴더 | 브랜치 |
|------|------|-----------|--------|
| 진다혜 | 인지부 (OpenCV, 1D-CNN) | `진다혜_인지부/` | `feature/인지부-진다혜` |
| 김승제 | 제어부 (STM32, CAN) | `김승제_제어부/` | `feature/제어부-김승제` |
| 정수영 | 통신부 (ESP-NOW, BLE) | `정수영_통신부/` | `feature/통신부-정수영` |
| 성현서 | 대시보드 (HMI, Nextion) | `성현서_대시보드/` | `feature/대시보드-성현서` |

---

## 브랜치 구조

```
main                    ← 최종 통합 브랜치 (직접 커밋 금지)
├── feature/인지부-진다혜
├── feature/제어부-김승제
├── feature/통신부-정수영
└── feature/대시보드-성현서
```

## 처음 시작할 때 (최초 1회)

> 노트북에 Git이 없으면 먼저 설치: https://git-scm.com/downloads

### 1단계. 레포 클론

```bash
git clone https://github.com/seungwp/-cdp.git
cd -cdp
```

### 2단계. 본인 브랜치로 이동

본인 이름에 해당하는 명령어 하나만 실행

```bash
git checkout feature/인지부-진다혜    # 진다혜
git checkout feature/제어부-김승제    # 김승제
git checkout feature/통신부-정수영    # 정수영
git checkout feature/대시보드-성현서  # 성현서
```

### 3단계. 본인 폴더로 이동

```bash
cd 진다혜_인지부    # 진다혜
cd 김승제_제어부    # 김승제
cd 정수영_통신부    # 정수영
cd 성현서_대시보드  # 성현서
```

여기까지 완료되면 개발 시작 준비 끝.

## 매번 개발할 때마다 (필수 루틴)

### 1. 작업 시작 전 — 최신 코드 받아오기

```bash
git checkout main
git pull origin main
git checkout feature/본인브랜치
git merge main
```

> main에 다른 팀원이 올린 변경사항이 있으면 내 브랜치에 반영하는 과정

### 2. 작업 — 본인 폴더 안에서만 수정

다른 팀원 폴더는 절대 건드리지 않기

### 3. 작업 완료 후 — 커밋 & 푸시

```bash
git add .
git commit -m "[역할] 작업 내용 요약"
git push origin feature/본인브랜치
```

커밋 메시지 예시:
```
[인지부] 차선 경계선 HSV 필터링 구현
[제어부] CAN 급제동 메시지 송신 테스트
[통신부] ESP-NOW 멀티홉 릴레이 로직 구현
[대시보드] 운전자 경보 UI 레이아웃 설계
```

---

## main에 반영할 때 (통합 시)

개인 작업이 완성됐을 때만 진행. 함부로 하지 않기.

```bash
git checkout main
git merge feature/본인브랜치
git push origin main
```

> 충돌(conflict)이 나면 혼자 해결하려 하지 말고 팀장한테 먼저 연락

## 주의사항

- `main` 브랜치에 직접 커밋 금지
- 본인 폴더 외 다른 폴더 수정 금지
- 작업 시작 전 반드시 `git pull` 먼저
- 커밋 메시지에 `[역할]` 태그 필수

## 모듈 간 인터페이스

각자 독립 개발하되, 아래 연결 지점은 사전 합의 후 개발

| 송신 | 수신 | 데이터 |
|------|------|--------|
| 인지부 → 제어부 | CAN 메시지 | 운전자 상태 (정상/위급), 전방 장애물 유무 |
| 제어부 → 통신부 | GPIO / UART | 위급 트리거 신호, 시나리오 선택 (A/B) |
| 통신부 → 대시보드 | UART / Serial | V2V 수신 데이터, 신호 강도, 패킷 수신율 |
| 인지부 → 대시보드 | Serial | 운전자 상태값, 주행 환경 정보 |
